# 검증 계획 · CareerPath AX for Counselors

> 골격 문서. 기능을 만들 때마다 영향 범위에 맞게 검증한다. 검증하지 못한 부분은 숨기지 않고 `미검증 범위`에 적는다. 실패한 테스트를 삭제·완화하지 않고 원인을 수정한다. (아래 섹션은 구현 착수 시 채운다 — 현재 미작성)

## 검증 항목
- 단위 테스트
- 통합 테스트
- 입력 검증
- 오류 상황 검증
- 경계값 검증
- 문서-코드 정합성
- README-기능 일치
- 제출물 / 배포물 구조

## 검증 방법과 도구
- 어떤 명령·도구로 어떻게 확인하는가

## 미검증 범위
- 아직 검증하지 못한 부분과 그 이유
