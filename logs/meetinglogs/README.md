# meetinglogs · 팀 회의록

회의록은 회의 1건당 **구조화 요약본 파일 1개**로 관리한다. Notion 녹음으로 만든 녹취록을
`meeting-notes` 스킬로 읽어 자동 생성한다. **녹취록 원문(받아쓰기 전문)은 git에 저장하지 않고
팀 Notion에서만 보관·공유한다**(개인정보·사담 보호). (규칙: CLAUDE.md §11.6)

## 파일 구조
```
logs/meetinglogs/
├── YYYY-MM-DD_<회의명>.md      ← 회의 1건 = 구조화 요약본 1개
└── README.md
```
- 파일 이름: `YYYY-MM-DD_<회의명>.md` (예: `2026-07-09_KDT-해커톤-기획-3일차.md`).
- 날짜는 Notion 녹취록 페이지의 **생성일(created_time, KST 기준)** 이다. 회의록을 만든 날이 아니다.
- 하루에 회의가 2건 이상이면 파일명으로 자연히 구분된다(회의명이 다르므로). 같은 이름이면 뒤에 `_2` 등을 붙인다.
- 원문 전사가 필요하면 요약 파일 상단의 **Notion 링크**로 접근한다(로컬에 원문 파일은 없다).

## 만드는 법 — `meeting-notes` 스킬
1. (첫 1회) `.claude/skills/meeting-notes/SKILL.md` 상단의 `NOTION_회의록_DB_URL`을 확인한다(이미 설정됨).
2. "회의록 만들어줘" / "녹취록 가져와" 라고 요청한다(또는 Notion 페이지 URL을 함께 준다).
3. 스킬이 Notion에서 최근 녹취록을 찾아 목록을 보여주면 하나를 고른다.
4. 스킬이 녹취록을 읽어 `YYYY-MM-DD_<회의명>.md`(구조화 요약)를 생성한다. 원문은 저장하지 않는다.
- 요약 형식·규칙: `.claude/skills/meeting-notes/summary-guide.md`
- Notion 커넥터 인증이 필요하다. 미인증이면 녹취록 텍스트를 직접 붙여넣는 폴백도 가능하다.

## 개인정보 규칙
- **원문 전사는 로컬/git에 남기지 않는다**(Notion 전용). 한 번 원격에 올라간 개인정보는 되돌리기 어렵다.
- 요약본에도 개인 신상·급여·연락처·사적 잡담 등 회의 외적 민감정보는 옮기지 않는다(CLAUDE.md §7).

## 결정·액션 처리 규칙
- 요약본은 회의에서 나온 **팀 결정 후보**와 **액션 아이템을 표시만** 한다.
- 스킬은 요약 후 발견한 결정·액션을 **보고**한다. 등록·관리는 사람이 한다:
  - 팀 전체 결정 → `logs/decisionlogs/_team.md`에 `D-NNN`으로 등록
  - 개인 결정 → 해당 팀원 `logs/decisionlogs/<이름>.md`(`D-<코드>-NNN`)
  - **액션 아이템(할 일) → 이 회의록 파일 안에서만 관리.** worklog에는 넣지 않는다(worklog는 완료 이력). 담당자가 그 일을 **완료하면** 그때 자기 `logs/worklogs/<이름>.md`에 `W-<코드>-NNN`으로 남긴다.
