## 안건별 논의 요약

### 1. 프로젝트 주제 선택 — PM 학습 가이드 vs 상담사 지원 서비스
- 두 후보를 비교. **PM 학습 가이드** 안은 우려가 컸다: (a) 가이드를 사람이 직접 만들어야 하면 확장성 부족(하드코딩), (b) 직무·회사마다 달라 통일된 가이드의 실효성 의문, (c) "LLM에 직접 묻는 것"·기존 강의 플랫폼과 차별화 부족.
- **상담사 지원 서비스**로 결정. 김윤서·박하린의 국취재(국민취업지원제도 상담) 경험에서 나온 문제 정의가 명확하다는 평가("문제 정리가 제일 깔끔하다"). 상담 전 상담사에게 구직자 맞춤 리포트를 제공해 전문적 상담을 돕는 방향.

### 2. 핵심 구현 기능
- **① 질문 리스트 생성**: 구직자의 실제 수준을 파악하는 질문 자동 생성 (예: "SQL 배웠다" vs "SQL로 실제 프로젝트 데이터 추출했다"의 차이 확인).
- **② 채용공고 점수화·추천**: 공고에서 스킬 키워드 추출 → 구직자 보유 스킬과 매칭. **TF-IDF**(문서 내 단어 중요도를 수치화하는 기법)로 점수화하고 **Cosine 유사도**(두 벡터가 얼마나 비슷한지 재는 값)로 적합도 계산.
- **③ 트렌드 분석**: 최근 공고와 3개월 전 공고의 키워드를 비교해 급증/급감 키워드 상위 ~5개를 상담사에게 제시.
- 추가 아이디어: 추천 근거에 **상담사 의견 반영**, **과거 상담 이력과 현재 상태 비교** 피드백.
- 기술 우려: TF-IDF가 기존 플랫폼 대비 단순·차별화 부족 우려 → 초안은 TF-IDF로 가되 부족하면 **AI 에이전트로 고도화**(재천 담당 언급). 포트폴리오 분석·NCS 능력단위는 **확장 기능**으로만 표기.
