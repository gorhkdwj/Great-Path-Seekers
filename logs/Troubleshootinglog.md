# Troubleshootinglog · CareerPath AX for Counselors (공용 단일 문서)

실제 오류·실패·환경 문제·검증 실패·설계 충돌이 발생하면 **이 하나의 문서**에 기록한다. 팀원 누구나 기록하며, **담당자(기록자)** 필드로 누구의 기록인지 명시한다. 다른 팀원의 로컬 에이전트도 여기서 해결책을 재사용할 수 있도록 **재사용 가능한 해결책**과 **후속 조치/대응 방안**을 반드시 채운다. (규칙: CLAUDE.md §11.4, §11.5)

## ⚠️ 문제 대응 순서 (반드시 먼저 수행)
1. 새 T-ID를 만들기 전에, 이 문서에서 **동일·유사 증상의 T-코드가 있는지 먼저 검색**한다.
2. 있으면 해당 항목의 **'재사용 가능한 해결책'을 우선 적용**한다.
3. 적용해서 해결되면 그 T-ID의 **'재발 이력'** 에 (작업자·날짜·적용 결과)를 덧붙인다.
4. 기존 해결책으로 해결되지 않거나 새로운 문제면 **새 T-ID를 생성**하고, 다른 팀원 에이전트가 재사용할 수 있도록 '재사용 가능한 해결책'과 '후속 조치/대응 방안'을 반드시 채운다.
5. 같은 문제가 반복되면 새 T-ID를 만들기 전에 기존 T-ID를 먼저 확인한다.

## 기록 형식
```
### T-00N · 문제 제목
- **담당자(기록자)**: 이름(코드) — 에이전트/사람
- **발생 상황**:
- **증상**:
- **확인된 원인**: (+ 불확실점)
- **조치 / 최종 해결**:
- **재사용 가능한 해결책**: (다른 팀원 에이전트가 동일 문제 시 그대로 적용할 수 있게 요약)
- **후속 조치 / 대응 방안**:
- **상태**: 미해결 / 해결됨 / 재발 관찰 중
- **재발 방지**:
- **재발 이력**: (다른 로컬에서 재발 시 — 작업자·날짜·적용 결과를 누적)
```
