# decisionlogs · 결정 기록

방향을 바꾸거나 이후 작업에 영향을 주는 **중요한 결정만** 기록한다. 사소한 수정은 worklog에만 남긴다. (규칙: CLAUDE.md §11.2, §11.3)

## 구성
- **개인 결정** → `logs/decisionlogs/<이름>.md` : 각 팀원이 자신의 작업에서 내린 결정. ID `D-<코드>-NNN`.
- **팀 전체 결정** → `logs/decisionlogs/_team.md` : 기술 스택, 구현 범위, 데이터 처리, 보안, 배포·제출 방식 등 팀 전체에 영향을 주는 결정. ID `D-NNN`(코드 없음).

## PRD §14 결정 기록의 위치
PRD(`docs/prd/careerpath-ax-planning.md`) §14의 D-001~D-013은 팀 공용 결정이 아니라 **김윤서·박하린의 주제 확정 브레인스토밍 판단 기록**으로, 두 사람의 개인 로그에 공동 결정으로 이관되었다(`김윤서.md` D-KYS-001~013 / `박하린.md` D-PHR-001~013, 동일 내용). PRD 원문은 기획 히스토리로 유지한다. `_team.md`의 D-NNN과 번호가 겹칠 수 있으므로, PRD 쪽 결정을 참조할 때는 **`PRD D-00N`** 으로 표기한다.

## 기록 형식
```
### D-<코드>-00N · 결정 제목        (개인)   /   ### D-00N · 결정 제목  (팀 공용 _team.md)
**상황** / **검토한 선택지** / **결정** / **근거** / **영향** / **재검토 조건**
```
