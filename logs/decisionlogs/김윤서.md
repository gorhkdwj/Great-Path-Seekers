## 이관 기록: 주제 확정 브레인스토밍 판단 (PRD §14에서 이관)

> 아래 D-KYS-001~013은 **김윤서·박하린 공동 결정**으로, 팀 주제 확정 브레인스토밍 과정의 판단 기록이다. 원문은 `docs/prd/careerpath-ax-planning.md §14`(PRD D-001~D-013)이며, 동일 내용이 `박하린.md`(D-PHR-001~013)에도 기록되어 있다. 원문 형식(결정/이유/영향)을 유지한다. 내용 수정 시 양쪽 파일을 함께 갱신한다.

### D-KYS-001 · 메인 사용자는 취업상담가로 확정 (공동: 김윤서·박하린 / PRD D-001)
- 결정: 서비스의 직접 사용자는 취업상담가로 한다.
- 이유: 지정주제의 "인간 역량 확장"을 구직자 개인 지원보다 더 선명하게 보여줄 수 있다.
- 영향: 기능 설계는 구직자 화면보다 상담가 리포트와 의사결정 보조 화면을 중심으로 한다.

### D-KYS-002 · 대상 구직자는 KDT 수료생/훈련생 기반 커리어 전환자로 한정 (공동 / PRD D-002)
- 결정: 일반 구직자가 아니라 KDT 커리어 전환자를 대상으로 한다.
- 이유: 대회 주최 맥락과 맞고, 문제 범위를 좁혀 차별화할 수 있다.
- 영향: 데이터와 시나리오는 KDT, HRD, 디지털 직무 전환 중심으로 설계한다.

### D-KYS-003 · 히어로 기능은 AI 상담 준비 리포트 자동 생성 (공동 / PRD D-003)
- 결정: 대표 기능은 상담가용 AI 상담 준비 리포트로 한다.
- 이유: 상담가의 역량 확장 장면이 가장 직관적으로 드러난다.
- 영향: 데모는 "구직자 입력 -> 상담 준비 리포트 생성 -> 상담 질문과 판단 근거 확인 -> 구직자 공유용 요약본 생성" 흐름으로 구성한다.

### D-KYS-004 · 유사 구직자 여정 추천은 확장 기능으로 둔다 (공동 / PRD D-004)
- 결정: MVP 핵심 기능에서는 제외하고 확장 기능으로 둔다.
- 이유: 실제 개인 여정 데이터는 공개 확보가 어렵고 개인정보 이슈가 크다.
- 영향: 예선 기획서에는 익명화 내부 데이터 연계 시 고도화 가능하다고 기술한다.

### D-KYS-005 · 기존 서비스와의 차별점은 상담가 중심 의사결정 보조로 잡는다 (공동 / PRD D-005)
- 결정: 원티드/자소설닷컴과 같은 구직자용 추천 서비스가 아니라 상담가용 AI 코파일럿으로 포지셔닝한다.
- 이유: 기존 서비스와 중복을 피하고, 해커톤 지정주제에 더 적합하다.
- 영향: 서비스명에 for Counselors를 포함한다.

### D-KYS-006 · 데이터 수집 전략은 공개 검색/다운로드 + 샘플 CSV 우선, API는 확인 후 연동 (공동 / PRD D-006)
- 결정: MVP에서는 고용24/워크넷 공개 검색 결과, 고용24 K-디지털 훈련 검색/목록다운로드, 샘플 CSV를 우선 사용한다. 채용정보 API와 훈련과정 API는 문서와 인증키 접근 가능성을 확인한 뒤 연동한다.
- 이유: 실제 공개 화면에서 채용공고와 K-디지털 훈련과정 데이터는 확인 가능하지만, API 문서와 인증키 접근은 추가 확인이 필요하다.
- 영향: 예선 기획서와 MVP 구현에서는 "API 자동 수집"을 핵심 전제로 두지 않는다. 데이터 확보 리스크를 낮추고, 구현은 CSV 기반 파이프라인으로 먼저 설계한다.

### D-KYS-007 · 점수는 합격 가능성 예측이 아니라 상담 참고용 적합도 점수로 정의 (공동 / PRD D-007)
- 결정: 공고 매칭 점수는 "합격 가능성"이 아니라 "공고 적합도 점수" 또는 "데모 매칭 점수"로 표현한다.
- 이유: 합격 여부를 예측하려면 실제 지원/합격 데이터가 필요하지만, 해당 데이터는 확보가 어렵고 개인정보 이슈가 크다.
- 영향: 스킬 일치도, TF-IDF 텍스트 유사도, 조건 적합도, 스킬갭 리스크 보정을 조합해 설명 가능한 추천 점수로 설계한다.

### D-KYS-008 · MVP 유사도 계산은 TF-IDF를 사용하고, 고도화 단계에서 임베딩으로 확장 (공동 / PRD D-008)
- 결정: MVP에서는 스킬 키워드 매칭과 TF-IDF 기반 cosine similarity를 사용한다.
- 이유: 구현이 빠르고 데이터가 적어도 동작하며, 상담사에게 추천 근거를 설명하기 쉽다.
- 영향: 의미 기반 유사도 한계는 인정하고, 고도화 단계에서 sentence-transformers 또는 OpenAI embeddings를 활용하는 방향으로 확장한다.

### D-KYS-009 · 최신 트렌드는 수집 시점의 공고 샘플에서 관찰된 키워드로 한정 (공동 / PRD D-009)
- 결정: 리포트에 최신 채용 트렌드를 반영하되, LLM의 일반 지식이 아니라 수집 시점의 공고 샘플에서 관찰된 요구 스킬과 직무 키워드로 정의한다.
- 이유: "요즘 트렌드"를 근거 없이 말하면 환각이나 과장으로 보일 수 있다.
- 영향: 리포트에는 "샘플 데이터 기준", "데모 공고 데이터 기준", "상담 참고용" 라벨을 붙인다.

### D-KYS-010 · 산출물은 구직자용 로드맵이 아니라 상담 준비 리포트로 구체화 (공동 / PRD D-010)
- 결정: CareerPath AX는 구직자에게 직접 정답을 제시하는 커리어 추천 서비스가 아니라, 취업상담사가 구직자의 상태와 시장 요구를 더 빠르게 이해하고, 상담 중 확인해야 할 질문과 판단 근거를 준비하도록 돕는 AI 상담 준비 리포트 서비스로 정의한다.
- 이유: 스킬갭, 로드맵, 대시보드만 강조하면 기존 구직자용 추천 서비스와 유사해 보일 수 있다. 상담 전 진단 브리핑, 상담사 확인 질문, 판단 근거와 불확실성, 상담 전략, 상담 후 기록을 포함해야 상담가의 역량 확장이라는 지정주제 적합성이 선명해진다.
- 영향: 히어로 기능에서 상담 질문 생성과 상담 준비 체크리스트를 강화하고, 출력물은 상담사용 내부 리포트와 구직자 공유용 요약본으로 분리한다.

### D-KYS-011 · 이전 상담 및 서류 변화 비교는 상담가용 핵심 보조 기능으로 둔다 (공동 / PRD D-011)
- 결정: 이전 상담 기록과 이전 이력서/포트폴리오 요약을 현재 자료와 비교해 구직자의 변화 상태, 개선된 부분, 남은 보완점을 상담사가 확인할 수 있게 한다.
- 이유: 상담사는 현재 상태만 보는 것이 아니라 이전 상담 이후 구직자가 실제로 무엇을 개선했는지 확인해야 더 구체적인 상담을 제공할 수 있다. 이는 구직자용 추천 서비스보다 상담가용 업무 보조 서비스라는 정체성을 강화한다.
- 영향: MVP에서는 데모용 이전/현재 요약 텍스트 비교를 사용하고, 화면은 변화 요약 표와 상담사 확인 질문을 함께 제공하는 방식으로 설계한다. 실제 개인정보 기반 상담 이력 저장과 원문 서류 버전 관리는 확장 기능으로 둔다.

### D-KYS-012 · 아이디어는 구현성·실효성·실용성 기준으로 계속 검토한다 (공동 / PRD D-012)
- 결정: 새 기능 아이디어는 기획서에 추가하기 전에 구현 가능성, 데이터 확보 가능성, 개인정보 리스크, 상담 현장 실효성, 기존 서비스와의 차별성을 함께 검토한다.
- 이유: 기능이 많아질수록 서비스가 구직자용 추천 서비스처럼 흐려지거나 해커톤 기간 내 구현 범위를 초과할 수 있다.
- 영향: 효과가 약하거나 리스크가 큰 아이디어는 MVP 핵심 기능이 아니라 확장 기능으로 분리하고, 필요한 경우 기능 표현과 범위를 수정한다.

### D-KYS-013 · 사회적 가치 축은 상담 품질 격차 완화로 둔다 (공동 / PRD D-013)
- 결정: CareerPath AX의 사회적 활용 가치는 지역별·기관별 상담 품질 편차를 줄이고, 상담사가 일정 수준 이상의 직무 맞춤 상담을 준비하도록 돕는 것으로 정의한다.
- 이유: 국민취업지원제도처럼 운영기관을 통해 상담이 제공되는 구조에서는 상담사의 경험, 기관의 운영 방식, 지역 자원에 따라 상담 품질이 달라질 수 있다. AI 상담 준비 리포트는 상담사를 대체하지 않으면서 공통 근거와 최신 직무 정보를 제공해 상담 품질의 기준선을 높일 수 있다.
- 영향: 문제 정의와 기대효과에서는 특정 기관의 상담 품질을 비판하기보다, 운영기관별 편차를 완화하고 상담 준비의 표준 근거를 제공한다는 표현을 사용한다.
