# Decisionlog · 팀 공용 (CareerPath AX for Counselors)

팀 전체 방향 결정을 기록한다. 대상: 문제 선정, 기술 스택 선택, 구현 범위 확정·축소, 데이터 처리 방식, 보안·개인정보 처리, 테스트·검증 방식, 배포·제출·공유 방식, 큰 구조 변경. (규칙: CLAUDE.md §11.3)

> 이 파일의 팀 결정은 **D-001부터 독립 번호**를 사용한다. PRD(`docs/prd/careerpath-ax-planning.md`) §14의 결정 기록 D-001~D-013은 팀 공용 결정이 아니라 **김윤서·박하린의 주제 확정 브레인스토밍 판단 기록**으로, 각자의 개인 결정 로그(`김윤서.md` D-KYS-001~013, `박하린.md` D-PHR-001~013)로 이관되었다. PRD의 결정을 참조할 때는 혼동을 피하기 위해 `PRD D-00N` 또는 `D-KYS-00N`/`D-PHR-00N`으로 표기한다.

## 기록 형식

```
### D-001 · 결정 제목
**상황** / **검토한 선택지** / **결정** / **근거** / **영향** / **재검토 조건**
```

### D-002 · 회의록 폴더 logs/meetinglogs/ 신설
**상황**
- 팀 회의(1~3차)가 진행 중이나 회의록을 기록할 표준 위치와 형식이 없었다. 김재천이 logs/ 하위 회의록 폴더 생성을 요청했다.

**검토한 선택지**
1. `meeting minutes` — 공백 포함, 기존 명명 패턴과 불일치
2. `meetinglogs` — 기존 `worklogs`·`decisionlogs`와 동일한 패턴(소문자·복수형·공백 없음)

**결정**
- `logs/meetinglogs/`로 생성. 파일 이름은 `YYYY-MM-DD_N차.md`, 회의 1건당 파일 1개.

**근거**
- 기존 폴더 명명 일관성 유지, 경로에 공백이 없어 도구·스크립트 취급이 안전하다.

**영향**
- CLAUDE.md·AGENTS.md에 §11.6(회의록 규칙) 추가, README 구조 갱신.
- 회의 중 팀 결정은 회의록에만 남기지 않고 `_team.md`에 D-ID로 기록 후 회의록에서 참조한다.

**재검토 조건**
- 회의록 양이 많아져 월별 하위 폴더 등 세분화가 필요해지는 경우.

### D-001 · 운영 로그를 logs/ 폴더 하위로 통합

**상황**

- 셋업 직후 루트에 `worklogs/`, `decisionlogs/`, `Troubleshootinglog.md`가 흩어져 있어 루트가 복잡했다. 김재천이 로그 관련 항목을 하나의 상위 폴더로 묶도록 요청했다.

**검토한 선택지**

1. 현행 유지(루트에 분산)
2. `logs/` 상위 폴더를 만들어 세 항목을 하위로 이동

**결정**

- `logs/` 폴더를 생성하고 `logs/worklogs/`, `logs/decisionlogs/`, `logs/Troubleshootinglog.md`로 이동한다.

**근거**

- 운영 로그가 한 폴더에 모여 루트가 단순해지고, 팀원·에이전트가 로그 위치를 한 곳에서 찾을 수 있다.

**영향**

- CLAUDE.md·AGENTS.md §3·§11·§11.5, README.md의 경로 참조를 일괄 갱신했다.
- `.gitignore`의 `logs/` 제외 패턴(런타임 로그용)이 팀 로그 폴더를 통째로 제외하게 되므로 해당 패턴을 제거했다(런타임 로그는 `*.log`와 `out/`으로 계속 제외).
- 모든 팀원 에이전트는 이후 `logs/` 하위 경로에 기록해야 한다.

**재검토 조건**

- 런타임 로그 폴더가 별도로 필요해져 이름 충돌이 생기는 경우.
